# Contributing

First off, thank you for considering contributing to this extension! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct (TBD).

## How Can I Contribute?

### Reporting Bugs

- Use the GitHub issues search to see if the bug has already been reported.
- If not, create a new issue. Be as specific as possible:
    - What were you doing?
    - What was the expected result?
    - What actually happened?
    - Include logs or error messages if possible.

### Suggesting Enhancements

- Check the issues list to see if someone else has already suggested it.
- Open a new issue with a clear title and description of the feature and why it would be useful.

### Pull Requests

1. **Fork the repo** and create your branch from `main`.
2. **Install dependencies**: Use `pnpm install`.
3. **Make your changes**: Ensure your code follows the project's style (TypeScript, clean architecture).
4. **Build the project**: Run `pnpm run build` to ensure everything compiles correctly.
5. **Update documentation**: If you're adding a feature, update the `README.md` and `CHANGELOG.md`.
6. **Submit the PR**: Link it to any related issues.

## Style Guide

- Use **Conventional Commits** for your commit messages.
- Keep functions small and focused.
- Use descriptive variable and function names.
- Document complex logic.
